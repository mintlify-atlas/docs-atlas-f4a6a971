## Technology Stack

Fluxora is built with modern, industry-standard technologies:

- **Backend**: Spring Boot 3.5, Spring Cloud, Netflix Eureka, Spring Security
- **Frontend**: Next.js 15, React 19, TypeScript, Tailwind CSS
- **Database**: PostgreSQL 15 with separate databases per microservice
- **Authentication**: JWT with role-based access control
- **Containerization**: Docker and Docker Compose
- **Cloud Deployment**: Render with managed PostgreSQL

## Architecture Overview

Fluxora follows a microservices architecture pattern with the following services:

- **API Gateway** (port 8080): Single entry point routing requests to microservices
- **Eureka Server** (port 8761): Service discovery and registration
- **Usuario Service** (port 8081): User authentication and management
- **Inventario Service** (port 8082): Inventory, products, recipes, and raw materials
- **Cliente Service** (port 8083): Client management and profiles
- **Entrega Service** (port 8084): Delivery tracking and route management
- **Frontend** (port 3000): Next.js application with role-based dashboards

Each microservice has its own PostgreSQL database, ensuring data isolation and independent scalability.

## Who Should Use Fluxora

Fluxora is designed for:

- **Manufacturing and Production Companies**: Track inventory, recipes, raw materials, and production batches
- **Delivery and Logistics Operations**: Manage routes, drivers, and real-time delivery tracking
- **Client-Centric Businesses**: Maintain comprehensive client profiles and delivery preferences
- **Teams Requiring Role-Based Access**: Separate admin, producer, and driver workflows
