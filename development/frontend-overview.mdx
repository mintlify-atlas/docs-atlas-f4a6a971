## Route Structure

### Authenticated Routes (`(app)` route group)

Protected routes requiring authentication:

- `/dashboard` - Admin dashboard
- `/dashboard/clientes` - Client management
- `/dashboard/inventario` - Inventory (Producer)
  - `/dashboard/inventario/productos`
  - `/dashboard/inventario/materias`
  - `/dashboard/inventario/recetas`
- `/dashboard/entregas` - Deliveries
  - `/dashboard/entregas/rutas`
  - `/dashboard/entregas/entregas`
  - `/dashboard/entregas/detalle-ruta`
- `/dashboard/repartos` - Distribution
- `/dashboard/reportes` - Reports
- `/dashboard/admin` - Admin tools

### Public Routes (`(auth)` route group)

- `/` - Login page

### Driver Routes

- `/driver` - Driver dashboard with map and deliveries

## Role-Based Routing

From `page.tsx`, users are redirected based on role:

```typescript
if (payload.role === "DRIVER") {
  window.location.href = "/driver";
} else if (payload.role === "PRODUCER") {
  window.location.href = "/dashboard/inventario";
} else {
  window.location.href = "/dashboard";
}
```

## API Integration

Frontend connects to backend through:

```typescript
const API_BASE = process.env.NEXT_PUBLIC_API_BASE; // http://localhost:8080
```

All requests go through the API Gateway on port 8080.

## Authentication Flow

1. User submits credentials on login page
2. POST request to `/api/usuarios/auth/login`
3. JWT token stored in localStorage
4. Token included in all subsequent requests
5. Role-based redirect to appropriate dashboard

## State Management

Uses React hooks and localStorage:
- `useState` for component state
- `useEffect` for side effects
- localStorage for authentication token
- No global state management library (keeps it simple)

## Development Setup

<Steps>
  <Step title="Install Dependencies">
    ```bash
    cd frontend-fluxora
    npm install
    ```
  </Step>
  <Step title="Configure Environment">
    Create `.env` file:
    ```bash
    NEXT_PUBLIC_API_BASE=http://localhost:8080
    ```
  </Step>
  <Step title="Run Development Server">
    ```bash
    npm run dev
    ```
    Opens on http://localhost:3000
  </Step>
</Steps>

## Build and Production

```bash
# Build for production
npm run build

# Start production server
npm start
```
