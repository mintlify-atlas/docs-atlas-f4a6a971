## Generating Reports

### Basic Workflow

<Steps>
  <Step title="Access Reports">
    Navigate to **Dashboard > Reports** from the main menu
  </Step>
  
  <Step title="Select Report Type">
    Click on one of the four report cards:
    - Deliveries Report
    - Sales Report
    - Inventory Report
    - Clients Report
  </Step>
  
  <Step title="Configure Filters">
    Set report parameters:
    - **Date Range**: Start and end dates
    - **Specific Filters**: Route, client, product (varies by report type)
  </Step>
  
  <Step title="Generate Report">
    Click **Generate Report** button
    - System processes data
    - Results appear in table format
    - Summary statistics displayed at top
  </Step>
  
  <Step title="Export (Optional)">
    Export to Excel or PDF:
    - Click **Export to Excel** for data analysis
    - Click **Export to PDF** for printing/sharing
  </Step>
</Steps>

## Delivery Reports

### Report Data

Delivery reports show:

- **Date**: Delivery date
- **Scheduled Deliveries** (entregasProgramadas): Total planned deliveries
- **Completed Deliveries** (totalEntregas): Successfully completed
- **Kg Corriente**: Regular products delivered (kg)
- **Kg Especial**: Premium products delivered (kg)
- **Total Kg** (kgTotal): Combined weight delivered
- **Completion %** (porcentajeCompletado): Success rate

### Summary Statistics

The report summary displays:

- **Total Deliveries Scheduled**: Sum of all planned deliveries
- **Total Deliveries Completed**: Sum of successful deliveries
- **Overall Completion Rate**: Average percentage completed
- **Total Kg Delivered**: Sum of all products distributed
- **Average Daily Deliveries**: Mean deliveries per day

### Filter Options

- **Date Range**: Select start and end dates
- **Route Filter**: View specific routes only
- **Driver Filter**: See deliveries by specific driver
- **Status Filter**: All, Completed, or Pending only

## Inventory Reports

### Report Data

Inventory reports show:

- **Product**: Product name
- **Type** (tipo): CORRIENTE or ESPECIAL
- **Initial Stock** (stockInicial): Stock at period start
- **Entries** (entradas): New production added (kg)
- **Exits/Waste** (salidas): Losses due to mermas (kg)
- **Final Stock** (stockFinal): Stock at period end
- **Total Value** (valorTotal): Inventory value in CLP
- **Turnover %** (rotacion): Inventory turnover rate
- **Waste %** (porcentajeMerma): Percentage of waste
- **Stock Status** (estadoStock): Healthy, Low, Critical

### Summary Statistics

- **Total Inventory Value**: Sum of all product values
- **Total Entries**: Total production volume
- **Total Exits**: Total waste/losses
- **Average Turnover**: Mean inventory rotation
- **Overall Waste Rate**: Total waste percentage

### Filter Options

- **Date Range**: Period for inventory analysis
- **Product Filter**: Specific product only
- **Category Filter**: panaderia, pasteleria, etc.
- **Status Filter**: Healthy, Low, or Critical stock levels

### Inventory Metrics Explained

<Accordion title="Turnover Rate (Rotación)">
  **Turnover % = (Exits ÷ Initial Stock) × 100**
  
  Measures how quickly inventory is sold:
  - **High turnover (&gt;80%)**: Good - products selling quickly
  - **Medium turnover (50-80%)**: Normal - steady sales
  - **Low turnover (&lt;50%)**: Concern - products moving slowly
  
  High turnover indicates:
  - Strong demand
  - Efficient inventory management
  - Fresh products
  
  Low turnover may indicate:
  - Overproduction
  - Declining demand
  - Need to adjust production
</Accordion>

<Accordion title="Waste Percentage (% Merma)">
  **Waste % = (Exits ÷ Total Production) × 100**
  
  Measures product loss:
  - **Acceptable waste (&lt;5%)**: Normal for bakery operations
  - **Moderate waste (5-10%)**: Monitor and improve processes
  - **High waste (&gt;10%)**: Critical - investigate causes
  
  Common causes of waste:
  - Expired products
  - Quality defects
  - Overproduction
  - Storage issues
  
  Reduce waste by:
  - Better demand forecasting
  - FIFO inventory practices
  - Quality control improvements
  - Proper storage conditions
</Accordion>

<Accordion title="Stock Status">
  System automatically categorizes stock levels:
  
  - **Healthy**: Stock above minimum threshold (green)
  - **Low**: Stock approaching minimum (yellow)
  - **Critical**: Stock below minimum or out of stock (red)
  
  Thresholds based on:
  - Historical daily demand
  - Production lead time
  - Safety stock requirements
</Accordion>

### Summary Statistics

- **Total Active Clients**: Number of clients with purchases in period
- **Total Purchases**: Sum of all transactions

- **Average Purchase per Client**: Mean transaction amount

### Filter Options

## Export Options

### Excel Export

Fluxora uses the **xlsx** library to generate Excel files:

**Excel Export Includes:**
- All report data in spreadsheet format
- Summary statistics sheet
- Formatted tables with headers
- Color-coded columns
- Formulas for calculated fields
- Charts and visualizations (future enhancement)

**When to Use Excel:**
- Need to perform custom calculations
- Creating pivot tables
- Combining with other data sources
- Detailed data analysis
- Sharing editable reports

<Steps>
  <Step title="Generate Report">
    Create the report with desired filters
  </Step>
  
  <Step title="Click Export to Excel">
    Click the green **Export to Excel** button
  </Step>
  
  <Step title="Download File">
    Browser downloads Excel file automatically
    - Filename: `reporte_[type]_[date].xlsx`
  </Step>
  
  <Step title="Open in Excel">
    Open file in Microsoft Excel or Google Sheets
  </Step>
</Steps>

### PDF Export

Fluxora uses **jsPDF** library to generate PDF reports:

**PDF Export Includes:**
- Company header with logo
- Report title and date range
- Summary statistics

- Page numbers
- Professional formatting

**When to Use PDF:**
- Sharing with stakeholders
- Printing reports

<Steps>
  <Step title="Generate Report">

  </Step>

## Tips & Tricks

- **Custom Date Ranges**: Use specific date ranges to analyze holiday periods or special events
- **Comparative Analysis**: Generate reports for same period last year to identify growth
- **Filter Combinations**: Combine multiple filters for detailed analysis (e.g., specific route + date range)
- **Export Templates**: Create Excel templates with formulas for recurring analysis
- **Scheduled Reports**: Plan to generate standard reports on the same day each week/month
- **Data Validation**: Cross-reference report data with delivery records to ensure accuracy
