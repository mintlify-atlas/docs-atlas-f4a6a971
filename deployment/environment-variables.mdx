## Overview

Fluxora uses environment variables to configure databases, service discovery, JWT authentication, and external integrations. Each microservice and the frontend require specific environment variables.

<Note>
All microservices use Spring Boot's `spring.config.import=optional:file:.env[.properties]` to load variables from `.env` files.
</Note>

## Variable Priority

Environment variables are loaded in this order (later takes precedence):

1. `application.properties` defaults
2. `.env` file in service root
3. System environment variables
4. Docker Compose `environment` section
5. Render environment variables

## Security Best Practices

<Warning>
Never commit `.env` files to version control. Always use `.gitignore`:

```gitignore
# Environment variables
.env
.env.local
.env.production
*.env
```
</Warning>

### JWT Secret Requirements

1. **Minimum Length**: 256 bits (32 characters)
2. **Randomness**: Use cryptographically secure random generation
3. **Consistency**: All microservices must use the **same** JWT secret
4. **Rotation**: Rotate secrets periodically in production

### Generate Secure JWT Secret

```bash
# Using OpenSSL
openssl rand -base64 32

# Using Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"

# Using Python
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

### Database Password Requirements

1. **Development**: Simple passwords OK (e.g., `fluxora123`)
2. **Production**: 
   - Minimum 16 characters
   - Mix of uppercase, lowercase, numbers, symbols
   - Use password managers or Render's auto-generation

### SMTP Password

<Warning>
NEVER use your actual Gmail password. Always use an App Password:
- More secure
- Can be revoked independently
- Required when 2FA is enabled
</Warning>

## Environment Variable Validation

### Missing Variables

If required variables are missing, services will fail to start with errors like:

```
Could not resolve placeholder 'DB_URL' in value "${DB_URL}"
```

### Validation Checklist

<Steps>
  <Step title="Check .env Files Exist">
    ```bash
    ls -la Microservicios/microservice-usuario/.env
    ls -la Microservicios/microservice-cliente/.env
    ls -la Microservicios/microservice-inventario/.env
    ls -la Microservicios/microservice-entrega/.env
    ls -la frontend-fluxora/.env
    ```
  </Step>
  
  <Step title="Verify Variable Names">
    Ensure variable names match exactly (case-sensitive):
    - `DB_URL` (not `DATABASE_URL`)
    - `EUREKA_URL` (not `EUREKA_SERVER_URL`)
  </Step>
  
  <Step title="Check for Trailing Spaces">
    Remove any trailing spaces or quotes:
    ```bash
    # Correct
    DB_PASSWORD=fluxora123
    
    # Incorrect
    DB_PASSWORD=fluxora123 
    DB_PASSWORD="fluxora123"
    ```
  </Step>
  
  <Step title="Validate JDBC URLs">
    Ensure proper JDBC URL format:
    ```bash
    # Correct
    jdbc:postgresql://host:port/database
    
    # Incorrect
    postgresql://host:port/database
    ```
  </Step>
</Steps>

## Docker Compose Overrides

Environment variables in `docker-compose.yml` override `.env` files:

```yaml
usuario-service:
  env_file:
    - ./Microservicios/microservice-usuario/.env
  environment:
    - TZ=America/Santiago  # Overrides TZ from .env
    - JAVA_OPTS=-Duser.timezone=America/Santiago
```

## Troubleshooting

### Variable Not Loading

1. Check file encoding is UTF-8 (not UTF-16)
2. Verify no BOM (Byte Order Mark) in file
3. Ensure Spring Boot version supports `.env` files (2.4+)
4. Check file permissions are readable

### Connection Refused Errors

**Symptom**: `Connection refused to eureka-server:8761`

**Solution**: Use internal service names in Docker:
```bash
# Correct for Docker Compose
EUREKA_URL=http://eureka-server:8761/eureka/

# Incorrect
EUREKA_URL=http://localhost:8761/eureka/
```

### JWT Token Invalid

**Symptom**: 401 Unauthorized errors

**Solution**: Ensure all services use the **same** `JWT_SECRET`:
```bash
# All services must have identical values
JWT_SECRET=your-secret-key-min-256-bits-change-in-production
```

## Complete Environment Setup

### Quick Start Script

Create all `.env` files at once:

```bash
#!/bin/bash

# Generate secure JWT secret
JWT_SECRET=$(openssl rand -base64 32)

# Usuario Service
cat > Microservicios/microservice-usuario/.env <<EOF
DB_URL=jdbc:postgresql://db-usuario:5432/usuario_db
DB_USERNAME=fluxora
DB_PASSWORD=fluxora123
SERVER_PORT=8081
EUREKA_URL=http://eureka-server:8761/eureka/
JWT_SECRET=$JWT_SECRET
JWT_EXP=60
EOF

# Cliente Service
cat > Microservicios/microservice-cliente/.env <<EOF
DB_URL=jdbc:postgresql://db-cliente:5432/cliente_db
DB_USERNAME=fluxora
DB_PASSWORD=fluxora123
SERVER_PORT=8083
EUREKA_URL=http://eureka-server:8761/eureka/
JWT_SECRET=$JWT_SECRET
JWT_EXP=60
EOF

# Inventario Service
cat > Microservicios/microservice-inventario/.env <<EOF
DB_URL=jdbc:postgresql://db-inventario:5432/inventario_db
DB_USERNAME=fluxora
DB_PASSWORD=fluxora123
SERVER_PORT=8082
EUREKA_URL=http://eureka-server:8761/eureka/
JWT_SECRET=$JWT_SECRET
JWT_EXP=60
EOF

# Entrega Service
cat > Microservicios/microservice-entrega/.env <<EOF
DB_URL=jdbc:postgresql://db-entrega:5432/entrega_db
DB_USERNAME=fluxora
DB_PASSWORD=fluxora123
SERVER_PORT=8084
EUREKA_URL=http://eureka-server:8761/eureka/
JWT_SECRET=$JWT_SECRET
JWT_EXP=60
SMTP_PASSWORD=your-gmail-app-password-here
EOF

# Frontend
cat > frontend-fluxora/.env <<EOF
NEXT_PUBLIC_API_URL=http://localhost:8080
NEXT_PUBLIC_APP_URL=http://localhost:3000
EOF

echo "Environment files created successfully!"
echo "JWT_SECRET: $JWT_SECRET"
echo "Don't forget to update SMTP_PASSWORD in Entrega service!"
```
