# Usuario Service

The Usuario Service is responsible for user authentication and management in Fluxora. It handles user accounts, roles, and JWT token generation.

## Responsibilities

- User authentication with JWT tokens
- User account management (CRUD operations)
- Role-based access control
- Password encryption with BCrypt
- Token generation and validation

## Entities

### Usuario Entity

The main user entity with the following fields:

- `id` (Long): Primary key
- `nombre` (String): User's full name
- `email` (String): Unique email address
- `password` (String): BCrypt hashed password
- `rol` (Rol): User role (ADMIN, PRODUCER, DRIVER)
- `createdAt` (LocalDateTime): Account creation timestamp
- `updatedAt` (LocalDateTime): Last update timestamp

### Rol Entity

Defines user roles:
- `ADMIN`: Full system access
- `PRODUCER`: Inventory management access
- `DRIVER`: Delivery management access

## Controllers

### AuthController

Handles authentication:
- `POST /api/usuarios/auth/login`: User login with JWT token generation
- `POST /api/usuarios/auth/logout`: User logout

### UsuarioController

Manages user accounts:
- `GET /usuarios`: List all users
- `POST /usuarios`: Create new user
- `GET /usuarios/{id}`: Get user by ID
- `PUT /usuarios/{id}`: Update user
- `DELETE /usuarios/{id}`: Delete user
- `GET /usuarios?rol={rol}`: Filter users by role

For detailed endpoint documentation, see [Usuario Endpoints](/api/endpoints/usuarios).

## Database Schema

The service uses PostgreSQL with two main tables:

**usuarios table:**
- id (BIGSERIAL PRIMARY KEY)
- nombre (VARCHAR)
- email (VARCHAR UNIQUE)
- password (VARCHAR)
- rol_id (BIGINT FK)
- created_at (TIMESTAMP)
- updated_at (TIMESTAMP)

**roles table:**
- id (BIGSERIAL PRIMARY KEY)
- nombre (VARCHAR)
