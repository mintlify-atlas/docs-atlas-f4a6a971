## Delivery Routes

### Get Active Routes

<RequestExample>
```bash cURL
curl -X GET https://api.fluxora.com/entrega/rutas-activas \
  -H "Authorization: Bearer {token}"
```
</RequestExample>

**GET** `/entrega/rutas-activas`

Retrieves all active routes with their assigned customers.

<Note>
  Requires **ADMIN** or **DRIVER** role authorization.
</Note>

#### Response

Returns an array of route objects with customer details.

<ResponseExample>
```json

  {
    "id": 1,

    "driverId": 5,
    "clientes": [
      {
        "id": 1,
        "nombreNegocio": "Panadería El Sol",
```

</ResponseExample>

#### Status Codes

- `200 OK` - Routes retrieved successfully

### Register Delivery

<RequestExample>
```bash cURL
curl -X POST https://api.fluxora.com/entrega/registrar \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "id_pedido": 123,
    "id_cliente": 1,
    "tipo": "VENTA",
    "productos": [
      {
        "id_producto": 1,
        "cantidad": 50
      }
    ]
  }'
```
</RequestExample>

**POST** `/entrega/registrar`

Registers a delivery to a customer.

<Note>
  Requires **ADMIN** or **DRIVER** role authorization.
</Note>

#### Request Body

<ParamField body="id_pedido" type="long" required>
  Order/session ID (required for VENTA type)
</ParamField>

<ParamField body="id_cliente" type="long" required>
  Customer ID
</ParamField>

<ParamField body="tipo" type="TipoMovimiento" default="VENTA">
  Movement type: VENTA (sale), DEVOLUCION (return), PRESTAMO (loan)
</ParamField>

<ParamField body="productos" type="array" required>
  Array of products delivered (required for VENTA)
</ParamField>

<Expandable title="productos object">
  <ParamField body="id_producto" type="long" required>
    Product ID
  </ParamField>
  <ParamField body="cantidad" type="double" required>
    Quantity delivered
  </ParamField>
</Expandable>

#### Response

<ResponseExample>
```json
{
  "message": "Entrega registrada exitosamente"
}
```
</ResponseExample>

#### Status Codes

- `200 OK` - Delivery registered successfully
- `400 Bad Request` - Invalid data or insufficient stock
- `403 Forbidden` - Insufficient permissions
- `500 Internal Server Error` - Technical error
