## Overview

As a **Driver**, you are responsible for executing daily delivery routes, managing customer deliveries, and recording delivery transactions. Your interface is optimized for mobile use during route operations.

<Tip>
Drivers have focused access to delivery features and client information. The driver interface is designed for on-the-go use.
</Tip>

## Key Responsibilities

Drivers manage:
- Daily delivery route execution
- Recording deliveries to clients
- Managing product inventory on vehicle
- Collecting payments
- Handling returns and exchanges
- Providing delivery summaries

---

<Steps>
  <Step title="Login to the System">

  </Step>
  
  <Step title="Review Daily Route">

  <Step title="Start Your Route">

## Understanding Your Route

### Route Assignment

Each driver is assigned to one delivery route by administrators:

```bash
Endpoint: GET /api/entregas/rutas/driver/{driverId}
Permission: ADMIN, DRIVER
```

Your route includes:
- Route name and identifier
- List of assigned clients
- Origin point (bakery location)
- Delivery schedule

### Route Optimization

Fluxora automatically optimizes your route:

<Card title="Smart Route Planning" icon="route">
  - Uses OR-Tools algorithm for optimal ordering
  - Minimizes total travel distance
  - Considers only clients with scheduled deliveries
  - Updates daily based on schedules
  - Uses real map data via OSRM
</Card>

### Viewing Today's Deliveries

Before starting, you can see:
- Total number of delivery stops
- Clients with scheduled deliveries
- Products and quantities to deliver
- Estimated route information

## Recording Deliveries

### Accessing Client Information

For each stop:

1. View client details:
   - Client name and address
   - Scheduled quantities (Corriente and Especial)
   - Delivery history
   - Route assignment

```bash
Endpoint: GET /api/clientes/cliente/{id}
Permission: ADMIN, DRIVER
```

### Making a Delivery

<Steps>
  <Step title="Select Client">
    From the "Entregas" tab, tap on the client you're visiting
  </Step>
  
  <Step title="Review Scheduled Order">
    The delivery form shows:
    - Programmed quantities for today
    - Product types (Corriente/Especial)
    - Client information
  </Step>
  
  <Step title="Record Actual Delivery">
    Enter what was actually delivered:
    - Adjust quantities if different from scheduled
    - Select delivery type (Sale, Return, Exchange)
    - Enter payment amount if collected
    - Add notes if needed
  </Step>
  
  <Step title="Submit Delivery">
    Tap "Submit" or "Registrar Entrega" to save
  </Step>
</Steps>

### Delivery Types

Fluxora supports different transaction types:

<Accordion title="VENTA (Sale)">
  Standard product delivery with payment:
  - Requires pedido ID (automatically set)
  - Must include products and quantities
  - Records payment amount
  - Deducts from vehicle inventory
</Accordion>

<Accordion title="DEVOLUCION (Return)">
  Client returning products:
  - Products returned to vehicle
  - May include credit or refund
  - Adds back to inventory
  - Documents reason for return
</Accordion>

<Accordion title="INTERCAMBIO (Exchange)">
  Exchanging one product for another:
  - Record what was returned
  - Record what was given
  - Net inventory adjustment
  - Note exchange details
</Accordion>

### Recording the Transaction

The system validates and records:

```bash
Endpoint: POST /api/entregas/entrega/registrar
Permission: ADMIN, DRIVER
```

Required information:
- Client ID
- Pedido ID (session ID)
- Delivery type
- Products and quantities
- Payment amount (for sales)

### Stock Deduction

When you record a sale:
1. System verifies sufficient stock
2. Deducts from oldest batches first (FIFO)
3. Updates real-time inventory
4. Confirms successful transaction

<Warning>
If stock is insufficient, the system will reject the delivery. Contact your administrator if this occurs.
</Warning>

## Managing Deliveries

### Viewing Delivery Progress

The header shows:
- Total pending deliveries
- Number completed
- Current client
- Time elapsed

### Skipping a Delivery

If a client is unavailable:
1. Note the skip reason
2. Continue to next client
3. Return later if possible
4. Undelivered items remain in your inventory

<Tip>
You can return to skipped clients later. The list shows both pending and completed deliveries.
</Tip>

### Viewing Delivery History

For any client, view past deliveries:

```bash
Endpoint: GET /api/entregas/entrega/cliente/{id}/historial
Permission: ADMIN, DRIVER
```

History includes:
- Delivery dates
- Products and quantities
- Payment amounts
- Delivery types
- Notes and comments

## Completing Your Route

### Finalizing Deliveries

When all deliveries are complete:

<Steps>
  <Step title="Verify All Recorded">
    Check that all clients show as completed or properly noted if skipped
  </Step>
  
  <Step title="Handle Returns">
    Record any returned products to adjust inventory
  </Step>
  
  <Step title="View Summary">
    The app automatically shows a summary when all deliveries are complete
  </Step>
</Steps>

### Route Summary

The end-of-route summary displays:

<Card title="Resumen de Ruta" icon="file-invoice-dollar">
  **Financial**:
  - Total dinero recaudado (cash collected)
  - Number of sales completed
  - Payment amounts
  
  **Products**:
  - Total kg Corriente delivered
  - Total kg Especial delivered
  - Comparison to starting quantities
  
  **Returns**:
  - Corriente returned
  - Especial returned
  - Return reasons
</Card>

### Summary is Saved

<Tip>
The route summary is automatically saved. You can view it again if you return to the app after completion.
</Tip>

The summary remains available until:
- You start a new route
- Administrator marks session as paid
- You clear it by clicking "Entendido"

## Client Management

### Viewing Client Details

Drivers can view (but not modify) client information:

```bash
Endpoint: GET /api/clientes/cliente/{id}
Permission: ADMIN, DRIVER
```

Available information:
- Client name and business name
- Delivery address
- Contact information
- Assigned route
- Delivery preferences
- Account status

### Client Notes

Add delivery notes for:
- Special instructions
- Access codes or directions
- Preferred delivery times
- Contact preferences
- Issues or concerns

<Warning>
Drivers cannot create, update, or delete clients. Contact your administrator for client changes.
</Warning>

## Troubleshooting

### Common Issues

<Accordion title="Cannot start route">
  Possible causes:
  - No deliveries scheduled for today
  - Route not assigned to your driver account
  - Previous session not properly closed
  
  Solution: Contact administrator to verify route assignment and schedules.
</Accordion>

<Accordion title="Delivery registration fails">
  Possible causes:
  - Insufficient stock in system
  - Missing required pedido ID
  - Network connectivity issues
  
  Solution: Verify stock levels and ensure you started the route session. Check internet connection.
</Accordion>

<Accordion title="Client not showing in route">
  Possible causes:
  - No scheduled delivery for today
  - Client not assigned to your route
  - Delivery already completed
  
  Solution: Verify schedule with administrator. Check completed deliveries list.
</Accordion>

<Accordion title="Map not loading">
  Possible causes:
  - GPS disabled on device
  - Network connectivity issues
  - Location permissions denied
  
  Solution: Enable GPS, check internet connection, grant location permissions to app.
</Accordion>

### Getting Unstuck

If you encounter technical issues:

1. **Check Basics**:
   - Internet connection
   - GPS enabled
   - Battery level
   - App permissions

2. **Try Recovery**:
   - Refresh the page
   - Log out and back in
   - Clear browser cache (if web app)
   - Restart device

3. **Contact Support**:
   - Call administrator
   - Provide error messages
   - Note which client/delivery
   - Continue manually if needed
