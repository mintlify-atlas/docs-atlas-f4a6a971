# Quickstart

Get Fluxora up and running on your local machine in just a few minutes using Docker Compose.

## Prerequisites

Before you begin, ensure you have the following installed:

<CardGroup cols={2}>
  <Card title="Docker" icon="docker">
    Version 20.10 or higher
  </Card>
  <Card title="Docker Compose" icon="layer-group">
    Version 2.0 or higher
  </Card>
  <Card title="Git" icon="git-alt">
    For cloning the repository
  </Card>
  <Card title="RAM" icon="memory">
    At least 4GB available
  </Card>
</CardGroup>

## Quick Start Guide

<Steps>
  <Step title="Clone the Repository">
    Clone the Fluxora repository to your local machine:
    
    ```bash
    git clone https://github.com/FelidevX/Fluxora.git
    cd Fluxora
    ```
  </Step>

  <Step title="Configure Environment Variables">
    Create `.env` files for each microservice with database credentials:
    
    <CodeGroup>
    ```bash Usuario Service
    # Microservicios/microservice-usuario/.env
    DB_HOST=db-usuario
    DB_PORT=5432
    DB_NAME=usuario_db
    DB_USER=fluxora
    DB_PASSWORD=fluxora123
    JWT_SECRET=your-secret-key-here
    JWT_EXP=60
    ```

    ```bash Cliente Service
    # Microservicios/microservice-cliente/.env
    DB_HOST=db-cliente
    DB_PORT=5432
    DB_NAME=cliente_db
    DB_USER=fluxora
    DB_PASSWORD=fluxora123
    JWT_SECRET=your-secret-key-here
    JWT_EXP=60
    ```

    ```bash Inventario Service
    # Microservicios/microservice-inventario/.env
    DB_HOST=db-inventario
    DB_PORT=5432
    DB_NAME=inventario_db
    DB_USER=fluxora
    DB_PASSWORD=fluxora123
    JWT_SECRET=your-secret-key-here
    JWT_EXP=60
    ```

    ```bash Entrega Service
    # Microservicios/microservice-entrega/.env
    DB_HOST=db-entrega
    DB_PORT=5432
    DB_NAME=entrega_db
    DB_USER=fluxora
    DB_PASSWORD=fluxora123
    JWT_SECRET=your-secret-key-here
    JWT_EXP=60
    ```

    ```bash Frontend
    # frontend-fluxora/.env
    NEXT_PUBLIC_API_BASE=http://localhost:8080
    ```
    </CodeGroup>

    <Note>
    Use the same `JWT_SECRET` across all services for proper authentication.
    </Note>
  </Step>

  <Step title="Start Services with Docker Compose">
    Launch all services using Docker Compose:
    
    ```bash
    docker-compose up -d
    ```
    
    This command will:
    - Pull required Docker images (PostgreSQL, Java, Node.js)
    - Build all microservices
    - Start 4 PostgreSQL databases
    - Launch Eureka Server, API Gateway, and all microservices
    - Start the Next.js frontend
    
    <Warning>
    First-time setup may take 5-10 minutes to download images and build services.
    </Warning>
  </Step>

  <Step title="Verify Services are Running">
    Check that all services started successfully:
    
    ```bash
    docker-compose ps
    ```
    
    You should see all services in "Up" state:
    - 4 PostgreSQL databases (ports 5433-5436)
    - Eureka Server (port 8761)
    - API Gateway (port 8080)
    - 4 microservices (ports 8081-8084)
    - Frontend (port 3000)
  </Step>

  <Step title="Access the Application">
    Open your browser and navigate to:
    
    <CardGroup cols={2}>
      <Card title="Frontend Application" icon="browser" href="http://localhost:3000">
        Main user interface
      </Card>
      <Card title="Eureka Dashboard" icon="server" href="http://localhost:8761">
        Service registry dashboard
      </Card>
      <Card title="API Gateway" icon="gateway" href="http://localhost:8080">
        API endpoint (for testing)
      </Card>
    </CardGroup>
  </Step>

  <Step title="Login to the System">
    Use the default credentials to log in:
    
    **Default Admin Account:**
    - Email: `admin@fluxora.uno`
    - Password: Contact your system administrator for the default password
    
    **Test the Login API:**
    ```bash
    curl -X POST http://localhost:8080/api/usuarios/auth/login \
      -H "Content-Type: application/json" \
      -d '{
        "email": "admin@fluxora.uno",
        "password": "your-password"
      }'
    ```
    
    You should receive a JWT token in the response.
  </Step>
</Steps>

## Troubleshooting

<Accordion title="Services not starting">
  **Port conflicts:**
  - Check if ports 3000, 8080, 8761, 8081-8084, 5433-5436 are available
  - Stop other services using these ports
  - Modify port mappings in `docker-compose.yml` if needed
  
  **Check logs:**
  ```bash
  docker-compose logs [service-name]
  ```
</Accordion>

<Accordion title="Database connection errors">
  - Ensure PostgreSQL containers are healthy:
    ```bash
    docker-compose ps
    ```
  - Check database logs:
    ```bash
    docker-compose logs db-usuario
    ```
  - Restart database services:
    ```bash
    docker-compose restart db-usuario db-cliente db-inventario db-entrega
    ```
</Accordion>

<Accordion title="Frontend not loading">
  - Check frontend logs:
    ```bash
    docker-compose logs frontend
    ```
  - Verify API_BASE environment variable is set correctly
  - Ensure API Gateway is running on port 8080
</Accordion>

<Accordion title="Authentication errors">
  - Verify all services use the same JWT_SECRET
  - Check that Usuario Service is running
  - Ensure Eureka Server is healthy
  - Verify user credentials exist in the database
</Accordion>

## Stopping Services

To stop all services:

```bash
docker-compose down
```

To stop and remove all data:

```bash
docker-compose down -v
```

<Warning>
The `-v` flag removes all database volumes. Use with caution in production.
</Warning>
